# Strategic Plan Exercise — Collaboration in a Box

A structured, git-native exercise for Data & Analytics Platform teams to develop, refine, and present their zone strategic plans — from first draft to executive-ready deck.

## How it works

```
Employee fills sections → Manager reviews & gives feedback → Deck is generated → Dry runs 1-2-3
```

Each team works on their own branch. Progress is visible in git. Claude Code acts as an always-available coach throughout.

---

## Roles

### Team Members
Your job is to work through the 8 section files in `teams/{your-team}/sections/` and fill them in with real, specific content. Claude Code will coach you as you go.

**Getting started:**
```bash
git checkout -b team/{your-team-name}
cd teams/example-team
# Copy this folder to teams/{your-team-name} and rename it
```

Work through sections in order. When you're ready for a check:
```bash
bash scripts/validate.sh teams/{your-team}
```

When all sections pass validation, generate your deck:
```bash
bash scripts/generate-deck.sh teams/{your-team}
```

### Managers
Your job is to review your team's work after each dry run stage and fill in the feedback files in `teams/{your-team}/feedback/`.

After each dry run, fill in the relevant stage file:
- `feedback/stage1-own-team.md` — after the safe-space run
- `feedback/stage2-peer-zone.md` — after the cross-zone run
- `feedback/stage3-leadership-preview.md` — after the pre-flight check

Commit your feedback to the team's branch so it's visible in git history.

### Executive Director
See `_executive/README.md` for your dashboard and interrogation guide.

```bash
bash scripts/dashboard.sh        # Status of all teams at a glance
bash scripts/dashboard.sh team/{name}   # Deep-dive one team
```

## Repo Structure

```
_executive/          # ED dashboard, talent notes, pattern analysis
_framework/          # Full framework reference and rubrics
scripts/             # validate.sh, generate-deck.sh, dashboard.sh
teams/
  example-team/      # Template — copy this for each real team
    sections/        # 8 section files (team fills these in)
    feedback/        # 3 feedback files (manager fills these in)
    deck/            # Generated Python script + .pptx (git-ignored)
    status.md        # Auto-generated by validate.sh
CLAUDE.md            # Coach instructions (Claude reads this automatically)
```

---

## The Three Tests

Every plan must pass before moving to dry run stage 3:

1. **Story Test** — Can you tell it in 3 minutes without slides?
2. **User Test** — Is every initiative traceable to real user pain?
3. **Investment Test** — Can you defend LOE against expected return?
